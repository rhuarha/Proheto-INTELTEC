==================================================
1. CONCEITO PRINCIPAL DO FLUXO DE PRODUÇÃO
==================================================

Toda Ordem de Produção obrigatoriamente deve passar pela etapa de Processamento.

A etapa de Processamento representa o momento em que o recebimento é processado no sistema CAPS e são informados:
- Produto
- Quantidade
- Multiplicador

Uma Ordem de Produção nasce no status "recebida", sem itens, e deve obrigatoriamente aparecer na tela de Processamento até que sejam incluídos um ou mais itens.

Somente depois de ter pelo menos um item com produto, quantidade e multiplicador, a Ordem de Produção poderá passar para o status "processada".

==================================================
2. NOVOS STATUS DA ORDEM DE PRODUÇÃO
==================================================

Substituir os status atuais da tabela producao pelos seguintes:

- recebida
- processada
- impressa
- envelopada
- embalada
- retirada
- cancelada

Definição dos status:

1. recebida
São as ordens de produção incluídas no sistema e que ainda não têm item nem quantidade. Estão aguardando processamento.

2. processada
São as ordens de produção que já receberam seus itens, produtos, quantidades e multiplicadores. Estão aguardando a próxima fase, que pode ser impressão ou envelopamento, conforme configuração do produto.

3. impressa
São as ordens de produção cujos itens que exigem impressão já foram impressos. Estão aguardando envelopamento ou embalagem, conforme configuração do produto.

4. envelopada
São as ordens de produção cujo envelopamento foi terminado. Estão aguardando embalagem.

5. embalada
São as ordens de produção já embaladas e disponíveis para retirada pelo cliente.

6. retirada
São as ordens de produção já retiradas pelo cliente.

7. cancelada
Ordens canceladas manualmente.

A atualização do status da Ordem de Produção deve ser feita automaticamente com base no status dos seus itens.

==================================================

Nome sugerido:
cliente_produto_preco

Campos:

- id
- cliente_id
- produto_id
- descricao
- preco decimal(10,2)
- data_inicial_validade date
- usa_papel enum('B', 'I')

- created_at

- deleted_at nullable

Significado do campo usa_papel:

- I = usa impresso próprio do cliente

Regras:
- O mesmo produto pode ter preços diferentes para clientes diferentes.

- Se houver preço futuro com data posterior à data_recebimento, ele não deve ser usado.
- Não permitir dois preços ativos para o mesmo cliente, produto e mesma data_inicial_validade.
- Ainda não implementar faturamento, mas a estrutura deve estar pronta para uso futuro.

8. TELA DE PROCESSAMENTO
==================================================

A tela de Processamento deve listar todas as Ordens de Produção com status "recebida".

- Cliente

- Hora de recebimento

Ao selecionar uma Ordem de Produção, abrir tela ou modal para incluir itens.

Campos do item:
- Item número: gerado automaticamente, não editável
- Produto: dropdown obrigatório
- Quantidade: número obrigatório
- Multiplicador: inteiro obrigatório default 1

Regras:
- Produto deve ser escolhido a partir do cadastro de produtos ativos para o cliente daquela ordem de produção.
- Ao incluir o primeiro item, a ordem ainda pode permanecer na tela até o usuário clicar em "Concluir Processamento".
- Permitir incluir múltiplos itens na mesma Ordem de Produção.
- Permitir remover item antes de concluir processamento.
- Ao clicar em "Concluir Processamento":
  - validar que existe pelo menos um item
  - validar que todos os itens têm produto, quantidade e multiplicador
  - alterar status da ordem para "processada"
  - registrar log
  - remover a ordem da lista de processamento

Validação de preço:
- Ao selecionar produto e concluir item, verificar se existe preço válido na tabela cliente_produto_preco para aquele cliente, produto e data_recebimento.
- Se não existir preço válido, exibir alerta claro:
  "Não existe preço válido para este cliente/produto na data de recebimento da ordem."
- Permitir ao usuário admin ou apontador cadastrar preço válido em modal.
- O preço cadastrado deve respeitar:
  - cliente_id da ordem
  - produto_id selecionado
  - data_inicial_validade menor ou igual à data_recebimento
- Após cadastrar o preço, permitir continuar o processamento.

13. REGRAS DE TRANSIÇÃO AUTOMÁTICA DE STATUS
==================================================

Criar uma função backend centralizada chamada recalcularStatusProducao(producao_id).

Ela deve verificar os itens da ordem e atualizar o status da produção conforme regras:

1. Se a ordem não tiver itens:
   status = recebida

2. Se tiver itens, mas ainda houver item incompleto:
   status = recebida

3. Se todos os itens estão cadastrados, mas não concluíram impressão/envelopamento/embalagem:
   status = processada

4. Se todos os itens que exigem impressão estão impressos:
   status pode avançar para impressa, salvo se já cumprir próxima etapa

5. Se todos os itens que exigem envelopamento estão envelopados:
   status pode avançar para envelopada, salvo se já cumprir próxima etapa

6. Se todos os itens estão embalados:
   status = embalada

7. Se todos os itens estão retirados:
   status = retirada

Importante:
- O status deve refletir o estágio mais avançado concluído pela ordem como um todo.
- Produtos que não exigem impressão ou envelopamento não devem bloquear avanço da ordem.
- Embalagem sempre é obrigatória.
- Retirada sempre é obrigatória para finalizar o fluxo.
